# Social Contracts

## Goal

Define party permissions, role behavior, and auditable social/economy administration actions.

## Rules

1. Party role constraints are explicit and enforceable.
2. Permission contracts remain synchronized with command and GUI behavior.
3. High-impact state changes are audit logged with actor/target context.
