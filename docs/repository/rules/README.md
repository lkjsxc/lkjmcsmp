# Repository Rules

## Goal

Define hard authoring and acceptance constraints for all changes.

## Rules

1. Docs-first ordering is mandatory for behavior changes.
2. Line limits are enforced by verification scripts.
3. Rule changes require explicit contract updates in this section.
